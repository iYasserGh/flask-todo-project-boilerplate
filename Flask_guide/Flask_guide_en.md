#### I’ve received several questions about how to continue your journey in Backend development using Flask. I’m answering them here so that everyone can benefit.


## First: There are two types of websites:

* **Type 1 (The one we learned together) is called Client-Side Rendering (CSR):**
    In this type, the two parts (Backend and Frontend) are separate. The page is rendered on the user's device using an API, just as we practiced.

* **Type 2 is Server-Side Rendering (SSR):**
    As the name suggests, the page is rendered on the server and then sent to the user fully prepared, without necessarily using an API.

Flask primarily focuses on the second type (SSR). Although you can build APIs and work with the first type, it is natively designed for the second. Therefore, when you see tutorials for Flask on YouTube, they will likely be teaching you how to build SSR websites.

## Third: Extensions
These are some extensions for the Flask library that improve your experience with SSR sites. You can research them and learn them step-by-step until you master website creation with Flask:

* **Databases:**
    [SQLAlchemy in Flask — Flask Documentation (3.1.x)](https://flask-sqlalchemy.palletsprojects.com/en/3.1.x/) (Prior knowledge of SQL basics is preferred).

* **Authentication and Login Processes:**
    [Flask-Login 0.7.0 documentation](https://flask-login.readthedocs.io/en/latest/)

* **Form Security and User Input Validation (e.g., login forms or specific fields):**
    [Flask-WTF — Flask-WTF Documentation (1.2.x)](https://flask-wtf.readthedocs.io/en/latest/)


## Finally: Is Flask the only and best option?
As mentioned, there are other libraries that might be better depending on your project. However, starting with Flask teaches you the fundamentals of the web, making it much easier to transition to other libraries later on.
